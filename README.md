# Anomaly Detection in Process Mining Logs Using Neural Networks

This project is developed to detect security and system anomalies using process mining log datasets. By analyzing user sessions (sandbox sessions) on system logs, suspicious or non-compliant behaviors are classified using machine learning and deep learning approaches.

The project incorporates both traditional Multi-Layer Perceptrons (MLP) and bidirectional recurrent neural networks (Bidirectional LSTM and Bidirectional GRU) designed for sequential data. Synthetic data generation methods have also been integrated to address the class imbalance problem.

## Dataset Structure and Attributes

The project loads and concatenates process mining logs from two different sources (`data/data1/process_mining.csv` and `data/data2/process_mining.csv`). The dataset contains the following main columns:

- **sandbox_id**: Unique identifier representing each isolated session.
- **cmd_name**: Name of the executed command.
- **cmd_arguments**: Arguments and parameters of the command.
- **cmd_type**: Category or type of the command.
- **level**: Security level or severity of the action.
- **timestamp_str**: Timestamp of the action (later parsed into a datetime object).
- **ip / hostname / username**: IP address, hostname, and username associated with the session.

## Feature Engineering

To train the models on raw log data, feature extraction was performed at two levels:

### 1. Session-Level Statistical Features (Tabular)
The following attributes were computed by grouping the log entries by `sandbox_id`:
- **session_duration**: Total duration of the session in seconds (calculated as the difference between the last and first log entries).
- **total_commands**: Total number of commands executed during the session.
- **unique_command_names**: Number of unique commands executed in the session.
- **command_entropy**: Shannon entropy of command frequencies (measures the complexity of the session).
- **unique_cmd_types / unique_ips / unique_hosts / unique_users**: The count of unique command types, IPs, hostnames, and usernames in the session.
- **avg_level**: Average security level of commands executed within the session.
- **commands_per_minute**: Average frequency of commands per minute.
- **average_time_between_commands**: The average duration between consecutive commands.
- **ratio_of_privileged_commands**: The ratio of privileged commands (commands with level >= 45) to total commands.
- **distinct_user_ratio**: Ratio of unique usernames to the total count of commands.

### 2. NLP (Natural Language Processing) Based Features
Command names and their arguments within each session were joined to build a text sequence. These sequences were processed using the **TF-IDF (Term Frequency-Inverse Document Frequency)** vectorizer to capture the semantic and word-based properties (top 20 terms) of each session.

## Anomaly Labeling Strategies

Since the raw dataset does not contain predefined labels, three unsupervised machine learning strategies were implemented and evaluated to label the dataset:

1. **95th Percentile Method**: A threshold-based method that labels the top 5% of sessions with the highest command counts as anomalies.
2. **Isolation Forest**: An tree-based algorithm that calculates anomaly scores based on how easily a data point can be isolated (contamination rate: 10%).
3. **Local Outlier Factor (LOF)**: A density-based algorithm that evaluates the local density of data points relative to their neighbors (contamination rate: 10%).

For the modeling phase, the labels generated by **Isolation Forest** were chosen as the primary target class due to its robust overall performance.

## Model Architectures and Classification Approaches

Five different model variations were trained and benchmarked:

### Multi-Layer Perceptron (MLP) Variations
An artificial neural network model trained on the concatenation of tabular statistical features and TF-IDF text vectors. The network consists of 64 and 32 hidden neurons with Batch Normalization and Dropout (0.3) layers, followed by a single output neuron with a Sigmoid activation.
- **MLP (Original)**: Trained on the original dataset without any class balancing techniques.
- **MLP (SMOTE)**: Trained on the training set balanced using SMOTE (Synthetic Minority Over-sampling Technique) to synthetically generate minority class instances.
- **MLP (ROS)**: Trained on the training set balanced using Random Over-Sampler (ROS).

### Sequential Deep Learning Models
Designed to detect anomalies by preserving the sequential order of commands within sessions. Command names are mapped to integer indices, padded to a fixed length (max_len=50), and passed through an Embedding layer.
- **Bidirectional LSTM (Long Short-Term Memory)**: Learns forward and backward temporal dependencies to capture anomalies in sequential command flows.
- **Bidirectional GRU (Gated Recurrent Unit)**: Similar to the LSTM architecture but utilizes gated recurrent units with fewer parameters.

## Installation and Execution

### Prerequisites

You can install all required dependencies by running the following command in your terminal:

```bash
pip install -r requirements.txt
```

The `requirements.txt` file includes:
- pandas
- numpy
- matplotlib
- seaborn
- scikit-learn
- tensorflow
- jupyter
- imbalanced-learn

### Training and Evaluating Models

To start the training pipeline:

```bash
python main.py
```

Running this command executes the entire lifecycle: data loading, preprocessing, feature extraction, labeling comparison, model training (MLP variants, LSTM, GRU), and saves all output metrics and charts to the `outputs/` directory.

---

## Evaluation and Comparison Results

Upon execution of `main.py`, all models are evaluated on the test dataset, and the following outputs are automatically generated:

- **Training History Charts**: Accuracy and loss curve plots per epoch (`outputs/*_accuracy.png`, `outputs/*_loss.png`).
- **Confusion Matrices**: Heatmaps visualizing classification errors (`outputs/*_cm.png`).
- **ROC Curves and AUC Scores**: Models' true positive rates vs. false positive rates (`outputs/roc_curves.png`).
- **Precision-Recall Curves**: Precision vs. Recall curves indicating model performance under class imbalance (`outputs/pr_curves.png`).
- **Model Comparison Summary**: A comparative summary table printed to the console and a comparison bar chart saved to files (`outputs/model_comparison.png`).
